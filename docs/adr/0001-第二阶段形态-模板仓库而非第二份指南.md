# 0001 第二阶段的形态:可克隆模板仓库 + 验证跑,而非第二份指南文档

第二阶段要与理论指南(《计算机视觉科研入门指南.pdf》)"相呼应",直觉方案是再写一份《实践指南》PDF。我们决定不写:第二阶段交付一个**可克隆的实践模板仓库**(README 使用文档 + 任务无关模板 + 作者验证跑作为活例子),读者围绕自己选的任务走闭环。理由:理论指南的哲学是"能力地图 + 贯穿例子",从不替读者做题;第二份 PDF 很容易退化成理论指南 §7 亲口警告的"复制命令堆工具",而模板仓库把"呼应"落成可执行的结构。

## Considered Options

- 第二份 PDF 实践指南(驳回:与理论指南重复演化,且命令式教程正是 §7 反对的形态)
- 只跑个人闭环不对外(驳回:与理论指南的"呼应"落空,§7 对读者的承诺不兑现)
- 指定任务的手把手教程(驳回:同 §7 理由;改为"框架 + 活例子 + 自选任务")

## Consequences

- 作者的验证跑本身是交付物的一部分:每一步都在验证 README 可被陌生人照做。
- README 的全部方法论只引用理论指南章节号,不复述(否则两个真相源)。
- 理论指南 PDF 需随仓库分发,否则章节引用悬空。
